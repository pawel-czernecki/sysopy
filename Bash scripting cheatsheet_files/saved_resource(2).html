                          Other File conditions:

                          [[ -b FILE ]] - True if the FILE exists and is a block special file.
                          [[ -c FILE ]] - True if the FILE exists and is a special character file.
                          [[ -G FILE ]] - True if the FILE exists and has the same group as the user running the command.
                          [[ -h FILE ]] - True if the FILE exists and is a symbolic link.
                          [[ -g FILE ]] - True if the FILE exists and has set-group-id (sgid) flag set.
                          [[ -k FILE ]] - True if the FILE exists and has a sticky bit flag set.
                          [[ -L FILE ]] - True if the FILE exists and is a symbolic link.
                          [[ -O FILE ]] - True if the FILE exists and is owned by the user running the command.
                          [[ -p FILE ]] - True if the FILE exists and is a pipe.
                          [[ -S FILE ]] - True if the FILE exists and is socket.
                          [[ -u FILE ]] - True if the exists and set-user-id (suid) flag is set.

                                  Actually this is wrong:

                                  $((RANDOM%=200)) # Random number 0..200

                                  As it keeps resetting RANDOM due to %= and eventually just generates two numbers over and over. The correct way to do this is:

                                  $((RANDOM % 200))

                                  Which actually produces numbers from 0..199.
